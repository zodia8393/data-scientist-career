# 문서 언어 정책

모든 신규 프로젝트는 기본 문서 언어를 **한글**로 작성합니다.

적용 범위:
- `README.md`
- 프로젝트 결과물(`final_report.md`, `model_card.md`, `data_source_and_contract.md` 등)
- 보고 및 추적 산출물의 텍스트 설명 영역

요구 원칙:
- 제목·설명·해설·품질 게이트 설명은 한글 기본으로 작성
- 수치, 코드, 파일 경로, 하이퍼파라미터 등은 기존 표기 유지
- 영문 템플릿이 필요한 경우, 최종 커밋 전 한글 버전으로 번역본을 추가

## 앞으로 프로젝트 적용 체크리스트
- 새 프로젝트 시작 시 `README_KR_TEMPLATE.md`를 복사해 `README.md` 초기본으로 사용
- 산출물 템플릿은 `.github/README_KR_TEMPLATE.md`, `.github/REPORTS_KR_TEMPLATE.md` 기준으로 작성
- 새 파일/문서는 기본적으로 한글 본문으로 작성하고, 영어는 코드·수치·명칭만 예외적으로 유지
- 마감 전 `README.md`와 최종 산출물 3개 문서(`final_report.md`, `model_card.md`, `data_source_and_contract.md`)를 최종 한글 검토
- 문체/품질 강제 기준:
  - 오직 한국어로 설명을 작성하고, 불필요하게 긴 문장/과장형 표현은 줄입니다.
  - 생성형 AI가 남기는 흔적이 나는 문장(예: 불필요한 공감문구, 과도한 반복, 형용사 나열, 의미 없는 서론/결론 장식문)을 제거합니다.
  - 작성은 실제 수행한 작업에 기반한 기술적 사실, 수치, 제약, 판단 근거 중심으로 쓰고, 개인 감상/과장 문구는 최소화합니다.
  - 문서 마지막에 반드시 `검토자 체크: 이 README/보고서가 사람이 쓴 문체로 읽히는가?(예/아니오)` 항목을 추가해 합의 기반으로 확인합니다.

## 용어 규칙
- `README.md`는 템플릿 문장보다 실제 기록형 문체를 우선시한다.
- 품질 게이트/실험 결과는 수치와 판별 근거를 먼저 제시한다.
